## 2. 验证方法

1. 在 `v0.0.1` worktree 中执行 `moon run cmd/main --target wasm`，捕获标准输出
2. 在 `HEAD` 中同样执行，JSON 格式输出包含全部 1681 个网格点的 `u`、`v`、`p`、`magnitude`
3. `v0.0.1` 输出包含：
   - 全场统计量（`max_u`、`max_v`、`max_p`、`min_p`）
   - 中心点完整数据（`u`、`v`、`p`、`|V|`）
   - 4 个采样点速度幅值（`|V|`）
   - 完整 CSV 格式全场数据（`x, y, u, v, |V|`，1681 行）
4. 用 Node.js 脚本逐点比对，容差 `tol = 1e-15`

## 8. 结论

**Chorin 求解器输出与 v0.0.1 完全一致。**

- 全部 1681 个网格点的 `u_final`、`v_final`、`|V|_final` 逐点差值为 **0**（浮点位级相同）
- 全场统计量 `max_u`、`max_v`、`max_p`、`min_p` 差值为 **0**
- 中心点 `u`、`v`、`p`、`|V|` 差值为 **0**
- SIMPLE 算法使用完全独立的全局状态数组（`g_u_s`、`g_v_s`、`g_p_s`），对 Chorin 求解器的 `g_u`、`g_v`、`g_p` 没有任何影响

测试 26（`simple_independent_of_chorin`）在代码层面已验证 SIMPLE 迭代前后 Chorin 中心点 `u` 值不变（容差 1×10⁻¹⁵），本报告从运行结果层面进一步确认了这一结论。
